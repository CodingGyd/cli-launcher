# 设置面板 + 自定义数据目录

## 背景

当前配置数据仅存储在浏览器 localStorage 中（Zustand persist），用户无法控制存储位置。需要增加一个设置入口（居中弹窗），支持自定义数据存储目录，并预留扩展空间。

## 方案概述

- 新增 `useSettingsStore` 管理设置项（`dataDir`），始终持久化到 localStorage
- 新增 `read_file` Rust 命令 + 文件同步层 `fileSync.ts`
- 设置自定义目录后，配置数据自动从 localStorage 迁移到指定目录的 JSON 文件
- 未设置目录时保持现有 localStorage 行为不变

## 实现步骤

### 1. Rust 后端：添加 `read_file` 命令
- `src-tauri/src/commands/mod.rs` — 新增 `read_file(path) -> Result<String>`
- `src-tauri/src/lib.rs` — 注册 `commands::read_file`

### 2. 前端服务层：添加 `readFile` 封装
- `src/services/tauri.ts` — 新增 `readFile(path)` 函数

### 3. 安装 shadcn Dialog 组件
- 运行 `npx shadcn add dialog`（base-nova 风格）

### 4. 创建 `useSettingsStore`
- 新建 `src/stores/useSettingsStore.ts`
- 字段：`dataDir: string | null`，`setDataDir` action
- 持久化到 localStorage key `cli-launcher-settings`

### 5. 给 `useConfigStore` 添加 `setItems` action
- `src/stores/useConfigStore.ts` — 新增 `setItems(items)` 用于从文件同步层注入数据

### 6. 创建文件同步服务 `src/services/fileSync.ts`
- `initFileSync()`：启动时检查 `dataDir`，如已设置则从文件加载配置覆盖 store，并启动订阅
- 订阅 `useConfigStore` 变化 → 当 `dataDir` 有值时自动写入 `dataDir/clilauncher-config.json`
- 切换目录时先将当前配置写入新目录

### 7. 创建 `SettingsDialog.tsx` 组件
- 新建 `src/components/SettingsDialog.tsx`
- 使用 shadcn Dialog 居中弹窗
- 内容分区：
  - **数据存储**：目录路径输入框 + 浏览按钮 + 重置按钮
  - **外观**：主题切换（深色/浅色）
  - 预留空间供未来扩展

### 8. 修改 `Header.tsx`
- 在主题按钮前添加齿轮图标按钮
- 点击打开 SettingsDialog
- 保留原有主题切换按钮（两处均可切换）

### 9. 修改 `App.tsx`
- `useEffect` 中调用 `initFileSync()` 初始化文件同步

## 验证

1. `cargo check` 通过
2. 启动 `npm run tauri:dev`，点击齿轮图标打开设置弹窗
3. 选择自定义目录 → 配置文件自动写入该目录
4. 重启应用 → 配置从自定义目录正确加载
5. 重置目录 → 回退到 localStorage 模式
